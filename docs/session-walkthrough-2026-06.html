01Project-centric Home for fabrication

Your Home now leads with the action you do most: starting a project.

What's there now

When you open Loglinkr as a plant head or admin, the Home page leads with a single teal card: "📐 Start a new project".

The Quick Entry grid below was trimmed to the actions you reach for most so the page is fast to scan instead of crowded.

How to use it

  1. Open Loglinkr.
  2. You land on Home. The big teal "Start a new project" card is right at the top.
  3. Supervisors and operators don't see this card (project creation is admin-only by design).
Status: Live
Audience: plant head & admin

02Creating a new project

A tight, non-technical-friendly form with seven fields.

What's there now

Tapping the "Start a new project" card opens a modal asking for: Project Name, Customer, Quantity / weight (free text — "12 MT" or "8 nos" both work), Target completion date, Leader, Manager, Supervisor(s) (pick any number).

The same person cannot be selected as both Leader and Manager — the dropdowns filter each other.

How to use it

  1. Tap Start a new project on Home.
  2. Type the project name. Pick a customer from your Masters (optional but recommended).
  3. Enter quantity, pick a target completion date.
  4. Choose the leader, manager, and tick the supervisors who'll work on this project.
  5. Tap Create project. You land back on the Projects tab.
Status: Live
Authorisation: plant head & admin only

03Home shortcuts — DC, PO, Upload Bill, Chat

One-tap deep links to the actions you do daily.

What's there now

Four new shortcut tiles at the top of the Quick Entry grid:

  • 📦 DC — opens Make Documents directly on the New DC form.
  • 🛒 PO — opens directly on the New PO form.
  • 📄 Upload Bill — jumps to Chat. If a chat group is already open, the file picker pops automatically so you can snap a bill immediately.

How to use it

  2. For Upload Bill: open the chat group first (the one with AI bill extraction enabled), then use the shortcut — the file picker appears immediately.
  3. If no chat group is open, the shortcut lands on the chat list and you pick a group first.

04Per-supervisor bill privacy

A supervisor sees only the bills they uploaded themselves. Cumulative views stay with the project manager, admin and plant head.

What's there now

When a supervisor opens the Documents tab on a fabrication plant, they see:

  • Bills they uploaded themselves.
  • Plant-wide bills that aren't tagged to any specific project.

They do not see bills uploaded by another supervisor — even on the same project. Expenses, payments and petty cash entries follow the same rule.

The project manager, admin and plant head see the cumulative view — every bill on every project, by every uploader. That's the financial visibility that keeps your project costs accountable.

How it works in practice

  1. A supervisor on Project A snaps a vendor bill in chat → tags it to Project A → sends.
  2. They see the bill in their Documents tab. Their teammate on the same project does not.
  3. The project manager, admin and plant head see both supervisors' bills, side by side, with vendor names and totals.
This rule is configurable. If you want a particular role to see all project bills instead of just their own, that's a setting in the roles matrix (Section 6). The default — "supervisor sees their own, project manager sees cumulative" — matches the way most fabrication plants want it.
Status: Live
Tables covered: bills, expenses, payments, petty cash
Configurable: yes, in roles matrix

05Tagging a bill to its project at upload time

A supervisor must tell the system which project the bill belongs to. Admins can leave it as a plant-wide bill.

What's there now

When an image is attached in a chat group that has AI bill extraction enabled, a teal panel appears above the input: "📐 Tag this bill to: [project dropdown]"

  • For supervisors, picking a project is required — the Send button won't fire until you choose.
  • For admins and project managers, leaving it blank is fine; the bill becomes a plant-wide entry.

The project tag travels from the chat attachment to the OCR pipeline, and the resulting bill is born tagged to that project. Your Documents tab will show it in the right project's records straight away.

How to use it

  1. Open a chat group that has AI bill extraction enabled.
  2. Tap the paperclip or camera and attach the bill photo.
  3. The "Tag this bill to" panel appears above the send button. Pick the project.
  4. Send. The bill is created, OCR runs, the project tag is applied automatically.
Status: Live
Required for: supervisors
Optional for: admins & project managers

06Custom roles & access matrix

A full walk-through of the roles system — built-in roles, custom roles, the permission matrix, and what each user sees.

Loglinkr now has its own roles-and-permissions system inside each plant. The five built-in roles (Plant Head, Admin, Manager, Supervisor, Operator) are still there, untouched — but you can also add custom roles like Purchase Officer or Site Engineer, and decide exactly which areas of the app each role can see and act on. This section walks through every screen.

6.1 · Where to find it

Inside People. A new "Roles" tab sits next to Units / Team / Operators / Shifts.

People tab → Roles sub-tab admin / plant head only

6.2 · The Roles list screen

The Roles tab lists your built-in roles, then any custom roles you've created, then a + New role button.

Roles list — built-in roles on top, custom roles below the divider, "+ New role" button at right.

6.3 · Creating a custom role

Tap + New role. A small modal asks for:

  • Display name — what users will see, e.g. "Purchase Officer".
  • Slug — the internal id, auto-derived from the display name. You usually don't need to change it.
  • Description — optional sentence to remind your team what this role is for.

After saving, the role exists with no permissions yet. The next step is to open the editor and tick what they can do.

6.4 · The permission matrix

The role editor is a grid: modules down the side, actions across the top. Tick the cells you want that role to be able to do. An "All" column on the right toggles every action for that module at once.

Modules are grouped for readability — Core & Communication / Shop Floor & Quality / Projects · Tasks · CAD / Commercial · Bills · Money / Customer & Quality Feedback / Reports & Audit.

Role editor — Purchase Officer with view+create on Documents, view on Stocks & Movements.

The six actions mean:

  • View — can see the area and its records
  • Create — can add new entries (e.g. upload a new bill)
  • Edit — can change existing entries
  • Delete — can remove entries
  • Approve — can mark records as approved (verified GRN, closed NCR, etc.)
  • Export — can download CSV/PDF of the records

6.5 · Worked example — a Purchase Officer

A common ask: someone who can upload vendor bills and see stock levels, but not see Sales documents, financial reports or the team. The full ticked set:

Result for the user with this role:

  • Their More menu shows Home, Chat, Stocks, Documents, Movements — and nothing else.
  • They can upload a vendor bill via OCR (View + Create on Documents).
  • They cannot open Sales DCs, Reports, People, or any audit screens.
  • They see only the bills they themselves uploaded (rule from Section 4).

6.6 · Assigning the role to a user

Roles only matter once you assign them. Go to People → Team. Each user row has a "👥 Roles" button. Tap it.

Per-user role assignment — Selvam K gets both Supervisor and Purchase Officer.

Multiple roles is fine. If you tick both Supervisor and Purchase Officer, the user gets the combined permissions. The more permissive wins. Keep everyone on Supervisor as a baseline and add custom roles to specific people.

6.7 · What the user sees after assignment

When the user signs in next, their More menu is filtered. Areas they don't have view on are simply absent — they don't see a "🔒 Locked" tile or any hint that the area exists.

More menu — supervisor's view (left) vs the trimmed view after replacing them with Purchase Officer (right).

6.8 · Safety nets

  • Plant Head and Admin can never be locked out of their own menu — those two built-in roles always have full access, by design.
  • If a user somehow ends up with zero permissions, the menu falls back to the standard view rather than showing an empty page.
  • The Plant Head and Admin built-in roles cannot be edited — they always carry all permissions. The other three built-in roles (Manager, Supervisor, Operator) are editable.
  • The "supervisor sees own uploads only" rule from Section 4 is the default. Want a custom role to see cumulative bills? Tick the View permission for Documents in their role; that role becomes a "cumulative" role.

07RDC (Returnable DC) with separate numbering

A second DC series, completely independent from regular DCs.

What's there now

A new + RDC sub-tab in Make Documents, next to + DC. RDCs:

  • Have their own document type (so they're distinguishable in lists and reports).
  • Have their own number series — completely independent from regular DCs.
  • Have their own prefix (e.g. MMPL/RDC/26-27/0156 while regular DCs continue as MMPL/DC/26-27/2347).

How to use it

  1. Make Documents → tap + RDC.
  2. First time only: set the starting number — see Section 9.
  3. Fill the form like a regular DC. Save.
  4. The next RDC goes +1; DCs continue independently.
Status: Live

08Delivery Challan first — Tax Invoice removed

Make Documents now focuses on DC / RDC / PO / Packing for fabrication plants.

What's there now

The + Invoice sub-tab inside Make Documents is no longer shown on fabrication plants. The visible sub-tabs are: + DC, + RDC, + PO, + Packing.

If a sale has previously been entered as an invoice, the record is still there — only the creation tab is hidden.

How it behaves

  1. Open Make Documents.
  2. You see DC / RDC / PO / Packing tabs.
  3. Invoice tab is no longer there.
Status: Live

09Continue numbering from your paper books

Set the starting number for any series so you don't restart from 1.

What's there now

At the top of the DC, RDC and PO creation forms, a small collapsible link: "▸ Set starting number for [type]". Visible to plant head and admin only.

Type the next number you want to issue (e.g. 2347), confirm, and the series continues from there for the rest of the financial year.

How to use it (your setup)

  1. Make Documents → + DC.
  2. Tap the "▸ Set starting number for DC" link.
  3. Enter 2347 (or whatever your next DC number is in your paper book).
  4. Confirm. The next DC issued will be MMPL/DC/26-27/2347.
  5. Repeat on the + RDC tab for the RDC starting number.
Status: Live
Authorisation: plant head & admin

10Password reset for a teammate

A plant head, admin or manager can issue a fresh login link without anyone outside your plant.

What's there now

On the People tab, each team-member row has a "🔑 Reset password" button. Tapping it generates a one-tap link that's valid for 24 hours.

A share modal opens with a copyable link and a green "Send via WhatsApp" button pre-filled to the user's registered phone number.

How to use it

  1. People tab → find the user.
  2. Tap 🔑 Reset password on their row.
  3. Tap Send via WhatsApp. The link goes to their phone.
  4. They tap the link → set a new password → auto-signed in.
Status: Live
Authorisation: plant head, admin, manager

113D model viewer inside Loglinkr

View STEP, STL, GLTF and other 3D files without leaving the app.

What's there now

A new tab "🧊 3D Models" in the Projects & Tasks group of the menu. Drag and drop a 3D file or use the file picker to load it.

The viewer has a ⛶ Fullscreen button — tap to expand to the whole screen. A circular × button at the top-right exits fullscreen.

How to use it

  1. More menu → Projects & Tasks → 3D Models.
  2. Drag and drop a STEP/STL/GLB file, or tap the file picker.
  3. Tap ⛶ Fullscreen for an immersive view; × or Escape to exit.
How to think about this: the 3D viewer is great for inspecting parts and taking measurements. When you've decided what to make, attach the final approved drawing / PDF back to the project record in Loglinkr so it lives with the rest of the project paperwork.
Status: Live

01Project-centric Home for fabrication plants

For: Marvel Machines and all fabrication-type plants. Other plant types see no change.

What's there now

When a plant head or admin opens Loglinkr on a fabrication plant, the Home page now leads with a single teal CTA card: "📐 Start a new project".

The Quick Entry grid below was trimmed from 10 buttons to 6 so the page feels purposeful instead of crowded. Sub-Contract, Stocks, Masters and Task moved out (still reachable via the menu).

How to use it

  1. Open Loglinkr on the /app URL.
  2. Land on Home. The big teal "Start a new project" card sits above Quick Entry.
  3. Operators / supervisors do not see this card (admin/plant-head only by design).
Status: Live
Scope: fabrication plants only
Audience: plant_head + admin

02"+ New Project" creation modal

A tight, non-technical-friendly form with seven fields. Skips Drawings/Notes (edit those after creation).

What's there now

Tapping the teal CTA opens a modal with: Project Name, Customer (from your Masters), Quantity / weight (free text — "12 MT" or "8 nos" both work), Target completion date, Leader, Manager, Supervisor(s) (multi-select).

The same person cannot be selected as both Leader and Manager. Membership is saved in a new mcp_project_members table, which powers per-project privacy in Section 4.

How to use it

  1. Tap Start a new project on Home.
  2. Type the project name. Customer dropdown is optional but recommended.
  3. Pick Leader, Manager, tick the Supervisor checkboxes.
  4. Tap Create project. You land back on the Projects tab.
Note: Quantity is stored in the project's notes field as "Qty: 12 MT". It's visible on the project page; it's not sortable or filterable yet. If you want a dedicated quantity column with units, that's a small follow-up.
Status: Live
Scope: fabrication plants only
Authorisation: server-side, plant_head/admin

03Home Quick-Entry shortcuts

One-tap deep links to the actions you do daily.

What's there now

Four new shortcuts at the top of the Quick Entry grid:

  • 📦 DC — opens Make Documents directly on the New DC form (not the list).
  • 🛒 PO — same idea, lands on New PO.
  • 📄 Upload Bill — jumps to Chat; if a chat group is already open, the file picker pops automatically so you can snap a bill immediately.
  • 💬 Chat — straight to the chat tab.

How to use it

  1. From Home, tap any shortcut.
  2. For Upload Bill: open the chat group first (the one with AI logistics enabled), then use the shortcut — the file picker appears immediately.
  3. If no chat group is open, the shortcut lands on the chat list and you pick a group first.
Status: Live
Scope: all plant types

04Per-project privacy for bills & expenses

Marvel's headline request. Genuinely enforced at the database, not just hidden in the UI.

What's there now

A supervisor in a fabrication plant who is a member of Project A only:

  • Sees Project A's bills, expenses, payments, petty-cash entries.
  • Sees plant-wide entries that aren't tagged to any project.
  • Cannot see bills, expenses, payments, or petty cash tagged to Project B — the database itself refuses to return those rows. Not just hidden in the UI.

Plant_head, admin and manager continue to see everything (not project-restricted, by design).

How to use it

  1. Create projects (Section 2) and assign supervisors as members.
  2. Bills get tagged to a project at upload time (Section 5).
  3. Assigned supervisors see only their projects' entries automatically. No further config needed.
Tested before shipping. Inside a rolled-back transaction we set up a synthetic Project A & B, a synthetic supervisor on A only, three test bills (A, B, plant-wide), and verified the supervisor saw exactly 2 of 3 and a plant_head saw all 3. Same test for expenses. Zero residual rows in production.
What this doesn't do (yet): field-level masking — i.e. hiding the vendor name or rupee total inside a bill the supervisor can already see. That's the next phase and is more architecturally involved. Production logs, NCRs, GRN receipts, jobwork lines are not project-gated yet — design call to start with bills & expenses where the customer asked.
Status: Live · DB-enforced
Scope: fabrication plants only
Tables gated: documents, expenses, payments, petty cash

05Project tag at OCR upload time

When a project-restricted supervisor uploads a bill, the system asks which project it belongs to.

What's there now

When an OCR-eligible image is attached to a chat group, a teal panel appears above the input: "📐 Tag this bill to: [project dropdown]"

  • For project-restricted users (supervisors), this is required — you can't send until a project is picked.
  • For admins and managers, it's optional — leaving it blank tags the bill as plant-wide.

The project_id flows from the chat attachment into the OCR queue, and a database trigger copies it onto the resulting bill document. So the bill is born with the right tag.

How to use it

  1. Open a chat group that has AI bill extraction enabled.
  2. Attach a bill photo (paperclip or camera).
  3. The "Tag this bill to" panel appears. Pick the project.
  4. Send. The bill is created, OCR runs, and the project_id is set automatically.
Status: Live
Picker shown when: RBAC plant + AI-enabled group + image attachment
Enforcement: required for restricted users

06Custom roles & module visibility (RBAC Phase 1)

For fabrication plants. Hide entire modules per role.

What's there now

A new Roles tab inside People (visible to plant_head + admin on fabrication plants). Lists five built-in roles (Plant Head, Admin, Manager, Supervisor, Operator) and lets the admin create custom ones — e.g. "Purchase Officer".

Each role gets a 27-module × 6-action permission grid (view, create, edit, delete, approve, export). Modules a user can't view are removed from their More menu.

The previous "👥 Roles" button on each Team row lets admins assign multiple roles to a user. Permissions are unioned.

How to use it

  1. People tab → Roles.
  2. Tap + New role, give it a display name (slug auto-derived), save.
  3. Tap Edit on the role, tick the modules + actions you want, save.
  4. Go back to Team → tap 👥 Roles on a user → tick the role(s) → save.
  5. That user's More menu now shows only allowed modules.
Phase 1 is visibility-only. Hidden modules disappear from the menu, but a determined user with browser developer tools could still query the API directly. Phase 2 closed this loophole for bills, expenses, payments and petty cash (Section 4) with real database enforcement. The remaining modules (production, quality, etc.) are still UI-visibility-only.
Status: Live
Scope: fabrication plants only
Safety net: plant_head/admin can never be locked out of their own menu

07RDC (Returnable DC) with its own number series

Marvel: "we issue regular DCs and returnable DCs separately."

What's there now

A new + RDC sub-tab in Make Documents, next to + DC. RDC has:

  • Its own document type (dc_out_returnable) so it's distinguishable in lists and reports.
  • Its own number series — completely independent from regular DCs. MARVELMA/RDC/26-27/0156 while DCs continue at MARVELMA/DC/26-27/2347.
  • A per-plant prefix (plants.rdc_prefix) so customisation is one SQL line.

How to use it

  1. Make Documents → tap + RDC.
  2. (First time only) set the starting number — see Section 9.
  3. Fill the form like any DC. Save.
  4. The next RDC issues at +1; DCs continue on their own track.
Status: Live
Scope: all plants (most useful for fabrication)
Tested: end-to-end with rollback

08Hide Tax Invoice tab on fabrication plants

Marvel said: "we don't bill via Invoice from this module."

What's there now

The + Invoice sub-tab inside Make Documents is hidden on fabrication-type plants. Other plant types (manufacturing, auto-IATF, jobwork, other) still see it.

How it behaves

  1. On fab plants: + DC, + RDC, + PO, + Packing visible. + Invoice hidden.
  2. Existing invoice documents (if any) still show in the recent list — only the creation tab is hidden.
Heads-up: the hiding is keyed on plant type, not a per-plant toggle. If a fabrication customer later asks for Invoice back, that's a separate small change (one new plant column + one if statement).
Status: Live
Scope: fabrication plants only

09"Set starting number" panel for DC / RDC / Invoice

Continue from your paper-book number, instead of restarting at 1.

What's there now

At the top of the DC, RDC and Invoice creation forms, a small collapsible link: "▸ Set starting number for [type]". Visible only to plant_head and admin.

Type the next number you want to issue (e.g. 2347), confirm, and from then on the series continues from there.

How to use it (Marvel's setup)

  1. Make Documents → + DC.
  2. Click the "▸ Set starting number for DC" link.
  3. Enter 2347 (or whatever the next DC is in Marvel's paper book).
  4. Confirm. Next DC issued will be MARVELMA/DC/26-27/2347.
  5. Repeat on the + RDC tab for the RDC starting number.
Status: Live
Authorisation: server-side, plant_head/admin only
Validation: 1 ≤ N ≤ 1,000,000

10Founder-side password reset (any user, any plant)

For when a plant head forgets and no other admin can help.

What's there now

Inside /admin, on the plant drawer's Team section, a "🔑 Reset" button next to each user. Tapping it generates a 24-hour, multi-use reset link.

A share modal opens with a copyable link and a "Send via WhatsApp" button pre-filled to that user's registered phone.

How to use it

  1. Sign in at loglinkr.com/admin as the founder.
  2. Find the plant card → drawer opens.
  3. Scroll to Team → tap 🔑 Reset on the user's row.
  4. Tap Send via WhatsApp. The link goes to their phone.
  5. They tap it → set a new password → auto-signed in.
Status: Live
Scope: platform admins only (founder)

11Admin-side WhatsApp reset link

A plant_head/manager can reset a teammate without the founder.

What's there now

Inside the regular People tab on the app, each team-member row has a "🔑 Reset password" button (admin / plant_head / manager can issue, for users other than themselves).

Same WhatsApp share flow as Section 10. The token is single-plant-scoped so a manager can only reset their own teammates.

How to use it

  1. People tab → find the user.
  2. Tap 🔑 Reset password on their row.
  3. WhatsApp opens, you tap Send, they reset.
Status: Live

12CadNexa 3D viewer embedded as a Loglinkr module

View STEP, STL, GLTF and other 3D files inside the plant.

What's there now

A new top-level tab "🧊 3D Models · CadNexa" in the Projects & Tasks More-menu group. Renders the CadNexa viewer inside an iframe at /cadnexa.

Has a ⛶ Fullscreen button — taps to a position:fixed overlay that covers Loglinkr's header and bottom nav. A circular × button (top-right, thumb-sized, respects iPhone notch) exits. Escape key works on desktop.

How to use it

  1. More menu → Projects & Tasks → 3D Models · CadNexa.
  2. Drag and drop a STEP/STL/GLB file, or use the file picker.
  3. Tap ⛶ Fullscreen for an immersive view; × or Escape to exit.
Heads-up: CadNexa has its own Supabase project. Files uploaded inside the embedded viewer live in CadNexa's storage, not Loglinkr's. The viewer is great for measuring/visualising; attach the final approved drawing/PDF back to Projects or NPD inside Loglinkr.
Status: Live
Scope: all plants

13Desktop More-menu fix

The "More" dropdown is now always fully visible.

What's there now

The desktop "More" dropdown is repositioned to clamp to the viewport. On smaller laptop windows, the bottom of the menu (Reports / Audit) is no longer cut off — the menu measures available space and scrolls inside itself if needed.

What changed

Replaced an absolutely-positioned dropdown (max-height 80vh measured against the document) with a fixed-position dropdown that, on every open/resize/scroll, measures the trigger's bounding rect and computes its own width, height, and horizontal anchor so it stays on-screen.

Status: Live
Scope: all plants (desktop only)
